# 項目表

## 目錄用途

- `worker-send/`：Cloudflare Email Worker 專案
- `docker-accept/`：接收 webhook 的 Node.js / TypeScript 服務
- `web-ui/`：issue #17 共用完整 HTML/CSS/JS UI，供 Docker 版與單文件版熱更新載入
- `single-file/`：issue #12 的 Python / Node.js / Java 單文件部署版本
- `.github/workflows/`：CI/CD，自動建置 Docker 映像
- `agent/`：本次接手紀錄與任務資訊

## 依賴概覽

### worker-send

- `wrangler`：Cloudflare Worker 開發與部署
- `typescript`：型別與編譯

### docker-accept

- `express`：HTTP API 與靜態頁面
- `typescript`：型別與編譯
- `tsx`：本地開發啟動
- 重要安全設定：`WEBHOOK_SHARED_SECRET` 用於 `/api/webhooks/email` 驗證，會先做固定長度 digest 再 constant-time compare；`MEMORY_EVENT_LIMIT` 控制記憶體模式事件保留數；未設定 `ADMIN_INITIAL_PASSWORD` 時會產生一次性 bootstrap 密碼並輸出到首次建立 admin 的啟動 log

### single-file

- `python/webhook_mail.py`：Python 標準庫 HTTP server，JSON 檔案持久化
- `node/webhook-mail.js`：Node.js 內建 `http` 服務，JSON 檔案持久化
- `java/WebhookMail.java`：JDK 內建 `HttpServer`，JSON 檔案持久化，內含輕量 JSON parser / writer

### web-ui

- `manifest.json`：UI 版本與檔案清單
- `index.html`：含 `{{TITLE}}`、`{{STYLE_CSS}}`、`{{APP_DATA_JSON}}`、`{{APP_JS}}` 佔位符的 HTML shell
- `style.css`：共用黑藍完整樣式
- `app.js`：用後端注入資料渲染登入頁與控制台

### GitHub Actions workflows

- `docker-accept-image.yml`：在 push / PR 時建置（push 時推送）`docker-accept` Docker 映像到 GHCR
- `publish-release.yml`：在 push `v*` tag 時自動建立 GitHub Release，並自動產生 release notes
